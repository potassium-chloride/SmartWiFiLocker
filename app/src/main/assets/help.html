Общие сведения

Приложение снимает и возвращает блокировку экрана устройства на основе данных WiFi. Теоретическая точность определения "безопасных" мест может достигать метра или даже меньше (зависит от настроек, планировки комнат, количества доступных WiFi точек). Что существенно лучше Google SmartLock, хотя не использует Trust API (которое недоступно сторонним разработчикам). При этом необязательно (хотя и желательно), чтобы устройство было подключено к WiFi сети. Программа стремится в любой непонятной ситуации защитить устройство паролем (цель приложения - сделать использование гаджета удобнее, не жертвуя безопасностью). Интернет для работы приложения не требуется.

Настройка точек

При создании точки происходит сканирование доступных сетей и запись их BSSID и соответствующих уровней сигналов. Название точки может быть любым, но не должно содержать символов перевода строки. Нужно только для удобства опознания, по умолчанию совпадает с SSID подключенной сети. Результат сравнения сохранённой точки с текущей вычисляется исходя из предположения, что уровень сигнала каждой точки почти подчиняется Гауссову распределению, выражается числом от 0 до 100. Ограничений на количество точек нет.

Порог доверия - значение, начиная с которого блокировка паролем снимается. Обычно 100% совпадения с записанной точкой не удаётся добиться. Рекомендуемые значения около 30-50.

Порог потери доверия - значение, начиная с которого блокировка возвращается. Должно быть меньше порога доверия (иначе точка бессмысленно занимает память). Обычно на 10-20 меньше чем порог доверия. Чем больше разница между порогами, тем инерционнее работает приложение.

Дисперсия уровня сигнала - дисперсия флуктуаций уровня сигнала точек доступа. Рекомендуемое значение от 2 до 4. Чем больше значение, тем больше радиус "беспарольной" зоны.

Число точек - число записанных WiFi сетей, которые используются для сравнения. Чем больше, тем лучше. Если число точек менее трёх, то однозначное подтверждение местоположения невозможно (беспарольная зона вырождается в две зоны, эллипс или кольцо)

Настройка метода

Общий. (KeyguardManager.KeyguardLock.disableKeyguard()) Не требует прав администратора, работает на устройстваах до AOSP 5.1 включительно. Не стабилен: если ОС выгружает фоновый процесс, то блокировка паролем возвращается (хорошо заметно на аппаратах с небольшим количеством ОЗУ при запуске тяжёлых приложений). Возможность случайного снятия пароля по ошибке исключена (скорее наоборот). Хоть включения администратора необязательно, всё равно рекомендуется включить, т.к. это защищает приложение от очистки данных или удаления.

Администраторский 1. (DevicePolicyManager.setDisableKeyguard()) Доступен начиная с AOSP 6.0. Как и при каких обстоятельствах работает выяснить не удалось. В теории должен быть похож на "общий"

Администраторский 2. (DevicePolicyManager.resetPassword()) "Механически" меняет пароль на устройстве. Для снятия блокировки выставляется пустой пароль, для возврата пароля выставляется записанный в память программы. Замечание 1: программа не может считать пароль с устройства, но может его изменить. Замечание 2: теоретически, если у телефона съёмная батарея и её извлечь в момент отсутствия пароля, то при включения можно получить доступ к устройству без пароля (воспроизвести ситуацию не удалось).
Если метод использовать невозможно, то настройка сбрасывается на "общий".
